# User Stories – Public Transport Platform

**Researcher:** Dinusha Amarasinghe 
**Date:** 2026-03-10 
**Branch:** `doc/dinusha-research`

---

## 1. Route Search

**User Story**

As a passenger, I want to search for bus or train routes, so that I can find the best way to travel to my destination.

**Acceptance Criteria**

- User can enter a starting location.
- User can enter a destination.
- System displays available bus or train routes.
- System shows estimated travel time.

---

## 2. Real-Time Arrival Information

**User Story**

As a commuter, I want to see real-time bus or train arrival times, so that I know when my transport will arrive.

**Acceptance Criteria**

- System displays next arrival times.
- Data updates automatically.
- User can refresh arrival information.

---

## 3. Journey Planning

**User Story**

As a user, I want to plan a journey between two locations, so that I can choose the fastest or most convenient route.

**Acceptance Criteria**

- User can input start and destination.
- System suggests possible routes.
- System shows estimated travel time.
- System displays transport types used (bus/train).

---

## 4. Transport Map View

**User Story**

As a passenger, I want to view transport routes on a map, so that I can understand how different routes connect.

**Acceptance Criteria**

- System displays routes on a map.
- User can zoom in and out.
- User can select routes to see details.

---

## 5. Service Alerts

**User Story**

As a commuter, I want to receive notifications about delays or cancellations, so that I can adjust my travel plans.

**Acceptance Criteria**

- System displays alerts for delays.
- System displays route disruptions.
- Notifications appear in the app interface.

---

## 6. Favorite Routes

**User Story**

As a regular traveler, I want to save my frequently used routes, so that I can access them quickly.

**Acceptance Criteria**

- User can mark a route as favorite.
- Favorite routes appear in a saved list.
- User can remove routes from favorites.

---

## 7. Tourist Friendly Information

**User Story**

As a tourist, I want transport information available in English, so that I can easily navigate the transport system in Sri Lanka.

**Acceptance Criteria**

- System interface is available in English.
- Transport instructions are clearly explained.
- Route and location names are readable.

---

## 8. Fare Information

**User Story**

As a passenger, I want to see the estimated fare for my journey, so that I can plan my travel budget.

**Acceptance Criteria**

- System displays estimated fare.
- Fare is shown before starting the journey.
- Fare updates when route changes.
