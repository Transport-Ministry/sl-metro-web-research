# 🚇 SL Metro Web Research

A collaborative research repository created as part of a Git training activity for IT students.
The findings collected here will directly inform the design and development of an official
website for the Sri Lanka Metro System.

## 📌 Purpose

This repository serves two goals:

1. **Technical** — Give students hands-on experience with Git branching, commits, and pull requests
   in a real team workflow.
2. **Research** — Collect references, feature analysis, and UI/UX observations from existing metro
   system websites around the world to guide the upcoming Sri Lanka Metro web project.

## 📁 Repository Structure

Each student works on their own branch following the naming convention `doc/firstname-research`,
containing two documents:
```
doc/firstname-research/
├── metro-research.md         ← Metro website research & brainstorm
├── university-curriculum.md  ← Student background & skill profile
└── screenshots/              ← Personal screenshots taken from reviewed websites
```

## 🌍 Research Scope

Students are reviewing public metro and transit websites from countries including Singapore,
United Kingdom, Japan, India, Germany, and Hong Kong — analysing features, UI/UX patterns,
and identifying what would work best in the Sri Lankan context.

## 🔀 Branch & Contribution Guidelines

- Branch naming: `doc/firstname-research` (lowercase, hyphens only)
- Commit format: `add: description - Name` / `update: description - Name` / `fix: description - Name`
- Do **not** push directly to `main`
- Open a **Pull Request** and wait for trainer review before merging
